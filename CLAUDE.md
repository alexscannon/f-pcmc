## Non-negotiable rules

- **One task per branch.** Branch name `task/TNN-short-name`. Do not mix tasks in one branch.
- **A task is done only when its listed tests pass AND `pytest` (full suite, including all prior tasks' tests) is green.** Never weaken, skip, mark xfail, or delete an existing test to make progress. If an existing test seems wrong, stop and report your reasoning instead of changing it.
- **Gate discipline.** Three tasks end in hard gates:
  - T6 `test_m1_gate` ‚Äî LTM-only routing must match batch knn_vmf AUROC ¬±0.01
  - T11 `test_golden_stream_end_to_end` ‚Äî executable spec of the full system (**GREEN as of 2026-07-11**; it reopened T9/T10 and found two real mechanism defects ‚Äî see the T7‚ÄìT11 bullets below. Keep it green: a later task that reddens it has broken a mechanism, not the gate.)
  - T14 `test_v1_regression_pin` ‚Äî ported v1 must reproduce pinned numbers A red gate means the implementation is wrong, not the gate. Do not proceed past a red gate. Fix the implementation ("fix the port, never the pin"), or stop and report if you believe the pin itself is wrong.
  - **The golden gate's rate clauses are q-linked** (owner amendments 2026-07-11). FR-5.1 puts œÑ at the q-th percentile of a concept's *own* LOO scores, so ~(1 ‚àí q/100) of a concept's own in-distribution arrivals fall beyond its own œÑ **by construction** (measured: 0.0569 at q=95). That is a hard ceiling of `q/100` on any "routed to the right concept at tier 1" rate and a hard floor of `1 ‚àí q/100` on any "still unknown" rate ‚Äî no implementation can beat it. The three gate clauses that measure such a rate therefore derive their bounds from `config.tau_percentile_q` ¬± a 3œÉ binomial band, **computed in the test, never hardcoded**. If you change `tau_percentile_q`, they re-derive themselves; if you write a new rate clause, respect the same ceiling.
- **No ground-truth leakage.** Nothing under `fpcmc/` may import from `eval/`'s ground-truth mapping or receive labels at runtime. The invariant tests enforce this; design accordingly.
- **No learning.** No torch autograd, optimizers, or model forward passes anywhere in `fpcmc/`. Embeddings are precomputed inputs.
- **Reference code is read-only.** `reference/stam` and `reference/pcmc` are populated git submodules (STAM, PCMC/`upl-benchmark`), pinned at fixed commits ‚Äî do not `git submodule update --remote` them or edit their contents. You may read them (and the source papers at `research_papers/`) to understand conventions, but nothing in `fpcmc/`, `eval/`, or `tests/` may import from `reference/`. `test_no_reference_imports` enforces this.
- **`lib/` is a frozen vendored snapshot, read-only.** Verbatim byte-identical copies from the source project, blob hashes recorded in `lib/PROVENANCE.md`. Never edit files under `lib/`; extract/adapt the math at the consuming site (`fpcmc/`, `eval/`) with citation comments pointing back to the lib file.
- **Determinism everywhere.** All randomness flows through `fpcmc/rng.py::make_rng(seed, stream="")` (named substream per module), seeded from config. No module-level RNG, no unseeded library calls (UMAP must receive `random_state`). `test_byte_determinism` and per-module determinism tests enforce this.
- **The golden stream fixture AND its run config are frozen.** `tests/fixtures/golden_stream.npz` + the pinned sha256 in `tests/fixtures/golden_stream.py` are gate-input data for T8/T11 ‚Äî never regenerate or re-pin without owner approval. A red golden gate is answered in `fpcmc/` ‚Äî never by editing a value in that file.
- **Specs are immutable.** Never modify anything under `docs/` or `tests/reference_numbers.yaml`. Propose spec changes in your report instead. Standing exception: **appending the completed task's entry to `docs/CHANGES.md`** (owner-directed changelog, 2026-07-10). ## Per-task workflow

1. Read the task's entry in the task plan and every PRD section it references. Read the tests it requires before writing implementation code.
2. Post a short plan (files to create/modify, test list, open questions). If any open question is a design decision not answered by the PRD, STOP and ask before implementing.
3. Write the task's tests first where practical (they are specified in the task plan); then implement until green.
4. Run: `pytest -m "not slow"` on every meaningful change; `pytest` (full, including slow/integration) before declaring the task done.
5. Commit in small logical units. Final commit message: `TNN: <summary> ‚Äî all tests green (<N> passed)`.
6. Append the completed task's entry to `docs/CHANGES.md` (what/why/commits/tests/approved deviations ‚Äî see the T0 entry for the format).
7. End the session with a report: what was built, test counts, runtime of integration tests vs NFR-1 budgets, any deviations proposed (never silently applied), and anything the next task should know.

## Stop conditions (halt and report rather than improvise)

- A gate test fails and the cause is not a clear implementation bug.
- A real-data schema check fails (`test_real_pool_schemas`).
- A pinned reference number appears unreachable within tolerance.
- The PRD is ambiguous on a decision that changes behavior.
- Any action would require editing `docs/`, `tests/reference_numbers.yaml`, or code under `reference/`.

- **T11 closed the golden gate, and three rules keep it closed. Do not weaken them without re-running the gate.**
  1. **Both merge guards are load-bearing.** FR-8.2 folds require `centroid_sim >= merge_sim` *in addition to* acceptance; FR-6 residual merges require `MergeSweeper.accepts_into`. Neither alone suffices: **the fold guard inspects the centroid but a fold transfers the REF_SET**, so a centroid-legitimate candidate carrying a contaminated ref_set poisons an LTM concept anyway (measured: `ltm_006` began with the tightest œÑ of all eight, 0.1387, and ran away to 0.7006 ‚Äî cohesion 0.29 ‚Äî capturing 169 known arrivals from seven other classes; with routing alone and no hooks it stays clean for all 2,000 steps, so the corruption is entirely hook-borne). Unguarded, T10 merged different-class candidates at centroid similarity down to **‚àí0.443**: the residual pool is a pool of OUTLIERS, and density clustering over outliers groups them *as outliers, not as classes* ‚Äî a shared HDBSCAN cluster is not evidence that two candidates are the same concept. **This is where contamination enters, so this is where it must be stopped.**
  2. **Merge admission is a DISJUNCTION, because `merge_sim` and œÑ measure different scales.** `merge_sim` is a **centroid-of-many** bar and only means what it says there (real data: same-class 20-sample half-centroids 0.94‚Äì0.98, cross-class 0.16‚Äì0.51). On **singletons** it is a category error ‚Äî a same-class *single* embedding scores only ~0.735 against its own class centroid, so a 0.8 bar refuses ~68% of genuine same-class singleton merges. œÑ is the singleton-scale instrument (it *is* a per-concept calibrated acceptance radius for single embeddings) ‚Äî but a seeded singleton's œÑ **is** the global prior (0.1537 ‚áí would demand sim ‚â• 0.846, stricter still). So `accepts_into` admits iff **a host with a calibrated œÑ (ref_set ‚â• 2) accepts the other's centroid, OR their centroids clear `merge_sim`**. Incoherent merges fail both legs; genuine consolidation passes at one scale or the other. **Never reuse `merge_sim` as a singleton bar, and never assume a singleton's œÑ is informative.**
  3. **`cohesion()` has ONE definition**, in `fpcmc/concepts.py` (a pure function of a ref_set), cached on `Concept` behind a lazy property invalidated on every ref_set mutation (`__setattr__` catches wholesale replacement ‚Äî merges; `add_observation` catches in-place reservoir replacement); `fpcmc.memory` re-exports the function. **FR-7 promotion is its only consumer.**
- **Do NOT gate routing on cohesion, and treat `min_cohesion` as uncalibrated on real data.** T11 briefly added a `cohesion >= min_cohesion` conjunct to tier-1 eligibility to stop the "black hole" (a candidate that absorbs many classes' œÑ-tail rejects earns a loose œÑ, and since the FR-4.3 margin is `(œÑ‚àís)/|œÑ|` a loose œÑ *raises* its margin, so it outbids the correct LTM). **Removed 2026-07-13** (owner ruling): the black hole was a *symptom* of the unguarded merges above ‚Äî with those guarded it never forms, and the whole suite incl. the golden gate passes without the conjunct. Worse, it would have been actively harmful on real data. **Measured on the real pools (64-member ref_sets): genuine single-class concepts span cohesion 0.165‚Äì0.708 (`ind_reference` 0.165‚Äì0.638) while mixed multi-class blobs reach 0.541 ‚Äî the populations OVERLAP, so no global threshold separates "one class" from "several".** CIFAR-100 class tightness varies ~4√ó.
